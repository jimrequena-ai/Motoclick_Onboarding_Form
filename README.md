# Motoclick Onboarding Form - Project Structure and Setup

Welcome to the Motoclick Onboarding Form project! This project has been built utilizing vanilla web technologies (HTML, CSS, JS) managed by the **Vite** bundler to make deployments to **Render** straightforward. And it connects directly to **Supabase** for storing our merchant entries securely.

## 📁 File Structure Explained

Here's an overview of our files and what each does. This is useful for any new team members getting started:

- **`index.html`**: The main interface file. Think of it as the skeleton of our application. All sections (A to F) are organized here with specialized classes matching our iOS-inspired design system.
- **`style.css`**: Our design system. It uses `CSS Variables` (`:root { --ios-bg: #F2F2F7; ... }`) extensively. This makes it effortless to change global colors (like turning a brand blue into another color). It's responsible for the premium iOS native look. 
- **`main.js`**: The brains of the form. It listens for the 'submit' click, collects the data from the HTML form elements, formats it perfectly as a `JSON Payload`, and calls our database client to save the information.
- **`supabase.js`**: The database connector. It initializes and manages the connection to Supabase securely utilizing the keys provided in `.env`.
- **`.env`**: (Created locally) Hidden environment variables file. It stores our sensitive Supabase URL and Keys. 
- **`package.json`**: The Node.js configuration file that tracks our installed dependencies (like `vite` and `@supabase/supabase-js`) and our command scripts (`npm run dev`).

## 🚀 How to Run Locally

If you are a newbie checking out the project, just follow these three steps in your terminal:

1. **Install Dependencies:**
   Run `npm install` inside the project folder. This downloads `vite` and `supabase-js`.
2. **Setup Supabase Keys:**
   Create a `.env` file referencing the actual keys (if you haven't yet, there's a `.env` created as an example).
   ```env
   VITE_SUPABASE_URL=https://your-project.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-key
   ```
3. **Start the Local Server:**
   Run `npm run dev`. This will open a fast live-reloading server, usually accessible at `http://localhost:5173`. Any change you make in the HTML/CSS will automatically refresh your browser!

## ⚙️ How to Deploy to Render

Render is incredibly easy for static assets. Follow these steps:

1. Push this entire repository to a Github/Gitlab repository.
2. Go to your Render Dashboard -> **New** -> **Static Site**.
3. Connect your repository.
4. Set the **Build Command** to: `npm run build`
5. Set the **Publish directory** to: `dist`
6. Click **Deploy**. Render will take care of building and hosting our iOS styled form onto a live URL automatically.

1. Go to Supabase and create a new project.
2. Using the SQL Editor, create a new table called `merchants`:
   ```sql

     id bigint generated by default as identity primary key,
     created_at timestamp with time zone default timezone('utc'::text, now()) not null,
     legal_name text not null,

     zip_code text,
     locations integer,
     address text,
     business_type text,
     operating_hours text,
     avg_orders integer,
     avg_ticket numeric,
     peak_hours text,
     own_drivers text,
     self_delivering text,
     using_3pl text,
     pain_points text[],
     delivery_platforms text[],
     pos_system text[],
     own_website text,
     own_app text,
     service_type text,

   ```
3. Now open `main.js` and uncomment the `supabase.from('merchants').insert([payload]);` code block once your table is ready. The form matches these column names exactly!
